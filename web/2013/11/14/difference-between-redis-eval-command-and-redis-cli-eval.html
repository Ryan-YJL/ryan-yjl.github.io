If you type redis-cli –help, you get the typical bullshit documentation.

It doesn’t offer any explanation why your arguments are not working, the way it should (or least the way you expected it to be like the EVAL command in interactive mode).

If you are using the command line format — redis-cli –eval, there is no ARGV variable in the file.

You only have KEYS, and can assess them like KEYS[1], KEYS[2], etc and you don’t have to specify the number of arguments like you do in the EVAL command.
